# 气味地图 —— 项目需求与方案文档

> 基于真实地图的社交化气味标记与视觉联觉平台  
> 整合日期：2026-06-11

---

## 一、核心概念

**产品名称**：气味地图（暂定）

**核心定位**：基于真实地图的轻社交 + 感官记录工具。以哈尔滨市地图为底，用户可以在地图上标记气味/心情，系统自动匹配颜色并以弥散光影效果呈现，不同气味之间颜色混合生成新的过渡色。

**一句话描述**：像发朋友圈一样，在地图上标记你闻到的气味（或心情），让它变成彩色光晕扩散出去。嗅觉和视觉进行"联觉"。

---

## 二、产品愿景

1. **趣味性优先**：这不是一个严肃的工具，而是一个像游戏一样好玩的社交应用
2. **极简操作**：哪怕是老人也可以操作，核心操作只有三步：点位置 → 输文字 → 确认
3. **社交属性**：每个用户都可以在地图上标记气味，不同的用户、不同的气味、不同的颜色，不同颜色之间可以混合成新的颜色进行过渡
4. **联觉体验**：用户随便输入一句话（如"刚吃了马迭尔的冰棍，真的好吃"），系统自动匹配颜色，实现嗅觉到视觉的联觉
5. **时间维度**：气味会随时间消散，保持地图的新鲜感

---

## 三、核心功能

### 3.1 气味标记

- 用户点击地图任意位置
- 弹出一个简洁的输入框
- 输入不超过 20 字的文字（可以是气味描述、心情、感受）
- 系统自动通过大模型将文字映射为颜色
- 确认后，该位置生成一个弥散光点

### 3.2 文字 → 颜色映射（核心机制）

**方案：大模型实时生成颜色**

不需要维护颜色数据库，不需要关键词匹配表。直接让大模型根据用户输入的任意文字返回一个颜色值。

**Prompt 设计**：
```
你是一个"文字到颜色"的转换器。
用户会输入一段简短的文字（最多20字），描述一种气味、心情或感受。
请根据这段文字的语义和情感，返回一个最能代表它的颜色。

规则：
1. 只返回一个 HEX 颜色值，格式如 #FF6B6B
2. 不要返回任何解释
3. 颜色要有美感，避免过于鲜艳或暗淡
4. 考虑中文语境和文化联想

用户输入：{用户文字}
颜色：
```

**示例映射**：

| 用户输入 | 返回颜色 | 理由 |
|----------|----------|------|
| "刚吃了马迭尔的冰棍，真的好吃" | #87CEEB | 冰凉的淡蓝色 |
| "闻到了丁香花的味道" | #DDA0DD | 淡紫色 |
| "今天心情很好" | #FFD700 | 明亮的金黄色 |
| "这条街好臭" | #556B2F | 暗沉的橄榄绿 |

**扩展参数**：可以让大模型同时返回颜色 + 视觉参数：
```json
{
  "color": "#HEX颜色",
  "saturation": "饱和度（心情强烈就高）",
  "brightness": "亮度（积极就高）",
  "diffusion": "扩散范围（浓烈气味小，清新气味大）",
  "opacity": "透明度（强烈就高）"
}
```

**成本分析**：
- GPT-4o-mini：约 $0.15 / 百万 tokens
- 一次请求约 200 tokens，单次成本约 $0.00003（几乎忽略）
- 响应时间约 500ms-1s

**缓存策略**：
- 相同输入 → 相同颜色（Redis 缓存）→ 一致性体验
- 热门输入（如"丁香花"、"烤红肠"）几乎不消耗 API 费用

**备选方案**：
- 本地小模型（Phi-3/Mistral 7B）：上线后大规模用户场景
- 确定性哈希（SHA256 → HEX）：零成本零延迟，但美感不可控

### 3.3 弥散光影效果

每个气味点不是简单的圆点，而是多层叠加的有机形态：

| 层级 | 描述 | 视觉特征 |
|------|------|----------|
| 第1层 | 核心亮点 | 小、亮、实，气味源头 |
| 第2层 | 内光晕 | 中等大小、半透明，气味浓的区域 |
| 第3层 | 外弥散 | 大、淡、噪声边缘不规则，气味飘散的范围 |
| 第4层 | 极淡余韵 | 最大、几乎透明，远处隐约的感觉 |

**关键技术**：
- GLSL 片段着色器实现径向渐变
- Simplex/Perlin 噪声让光点边缘不规则波动，模拟气味扩散的自然感
- WebGL 加法混合（Additive Blending）让不同颜色重叠区域产生混色效果

### 3.4 颜色混合

当不同用户标记的气味光点重叠时，自动混合：

```
红色（烤红肠）+ 绿色（丁香花）= 黄色过渡带
蓝色（江风）+ 红色（夜市）= 紫色过渡带
```

### 3.5 缩放适配

| 缩放级别 | 视觉表现 |
|----------|----------|
| 城市全景（zoom 10） | 光点缩小为小色块，整体像一幅水彩画，颜色柔和交融 |
| 街区级别（zoom 14） | 光点中等大小，能看到单个气味源的光晕形状，边缘有噪声纹理 |
| 街道级别（zoom 17） | 光点变大，弥散效果明显，能看到呼吸动画和颜色混合细节 |

### 3.6 时间消散

- 新标记：光点亮、饱和、呼吸明显
- 1小时后：开始变淡，扩散范围缩小
- 24小时后：很淡，像一层薄雾
- 7天后：几乎消失，只留一丝痕迹

## 四、技术方案

### 4.1 技术栈

| 模块 | 技术选型 | 说明 |
|------|----------|------|
| 地图底图 | Mapbox GL JS | 哈尔滨底图，支持缩放，原生 setFog() 大气迷雾 |
| 光点渲染 | deck.gl ScatterplotLayer + 自定义 Shader | WebGL2 高性能渲染，支持加法混合 |
| 颜色生成 | OpenAI API（GPT-4o-mini）| 文字 → 颜色映射 |
| 前端框架 | 原生 HTML/JS 或 React | 按需求选择 |
| 数据存储 | Firebase / Supabase | 实时同步光点数据 |
| 缓存 | Redis | 缓存已生成的文字→颜色映射 |

### 4.2 为什么选 deck.gl + Mapbox

1. deck.gl 基于 WebGL2/WebGPU，百万级数据点流畅渲染
2. 加法混合模式天然支持发光效果，密度越高越亮
3. 与 Mapbox 摄像机完美同步，所有缩放级别一致
4. 支持自定义 Shader，可实现任意视觉效果
5. Mapbox 内置 setFog() 方法，可直接添加大气迷雾

### 4.3 备选方案

| 方案 | 适用场景 | 劣势 |
|------|----------|------|
| Three.js + Mapbox | 更复杂的 3D 体积光/体积雾 | 集成复杂度高，性能开销大 |
| 纯 Canvas 2D | 简单原型、少量数据点 | 无法处理大量数据 |
| Kepler.gl | 快速探索数据 | 定制性有限 |

## 五、气味地图领域参考

### 5.1 Kate McLean — Sensory Maps（感官地图）
- 核心人物，专注气味地图研究 12 年以上
- 方法：组织"气味漫步"（Smellwalk）实地采集数据
- 代表作：Smellmap: Pamplona（粒子动画呈现气味飘散）
- 网站：https://sensorymaps.com/

### 5.2 Smelly Maps — MIT/剑桥大学
- 用社交媒体数据（Flickr、Instagram、Twitter）推断城市气味
- 建立首个城市气味词典，10 大分类
- 覆盖伦敦、巴塞罗那、纽约等 12+ 城市
- 在线地图：http://goodcitylife.org/smellymaps
- 论文：https://arxiv.org/pdf/1505.06851

### 5.3 What Does Boston Smell Like? — Tufts University
- 受 Kate McLean 理论启发
- 动态数据驱动地图
- 链接：https://www.informationisbeautifulawards.com/showcase/7273

### 差异说明
上述项目都是**静态研究地图**，气味地图是**实时社交应用**——用户可以自己标记、实时互动、颜色混合、时间消散，产品形态完全不同。
